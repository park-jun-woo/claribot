## Claritask 프로젝트 개요

### 한 줄 요약
**LLM 기반 프로젝트 자동 실행 시스템** - Claritask가 오케스트레이터, Claude Code가 실행기

### 핵심 목표
| 목표 | 설명 |
|------|------|
| 자동화 | 프로젝트 수동 세팅 자동화 (30-50분 절약) |
| 무제한 작업 | Task 수 제한 없이 무인 작업 |
| 컨텍스트 극복 | 매 Task마다 컨텍스트 초기화 |

### 아키텍처 (2-Phase + TTY Handover)

```
clari init → Claude Code [Phase 1: 요구사항 수립]
                 ↓
           clari project start
                 ↓
           Claude Code [Task 1] → [Task 2] → [Task N]
                 ↓
           최종 보고
```

### 핵심 개념

1. **FDL (Feature Definition Language)**
   - YAML로 기능 계약 정의
   - Python으로 코드 스켈레톤 자동 생성
   - LLM은 TODO만 채움 (함수명/타입/API 경로는 확정)

2. **그래프 기반 데이터 구조**
   - `project → feature → task` 계층
   - Edge로 의존성 관리
   - Topological Sort로 실행 순서 자동 결정

3. **TTY Handover**
   - Claritask가 Claude Code에게 터미널 제어권 위임
   - 코딩 + 테스트 + 디버깅 완전 자동화

### 기술 스택
- **CLI**: Go + Cobra + SQLite
- **스켈레톤 생성**: Python
- **확장**: VSCode Extension

### 프로젝트 구조
```
claritask/
├── cli/              # Go CLI 소스코드
├── vscode-extension/ # VSCode 확장
├── specs/            # 요구사항 명세서
├── features/         # FDL 파일들
├── tasks/            # 해야할 Task 문서
└── finished/         # 완료한 Task 문서
```
